Section 1

The "Holes" in $\mathbb{Q}$

Why do we need the Real Numbers ($\mathbb{R}$)? Because the Rational Numbers ($\mathbb{Q}$) are full of "holes".

Consider the sequence of rational numbers $1, 1.4, 1.41, 1.414, \dots$ which approaches $\sqrt{2}$. Even though every entry is in $\mathbb{Q}$, the limit is not. $\mathbb{Q}$ doesn't have enough "stuff" to hold limits.

The Dedekind Cut Intuition

Richard Dedekind proposed a way to "construct" the real numbers using the rationals. Imagine cutting the rational number line into two sets, $A$ and $B$, such that every element in $A$ is less than every element in $B$.

If there is no largest element in $A$ and no smallest element in $B$, that "cut" represents an irrational number—a gap where a real number *should* be.

The Analogy

If $\mathbb{Q}$ is like a beaded necklace, $\mathbb{R}$ is like a smooth, solid wire. Between any two beads, there's always space, but the wire is continuous. In $\mathbb{Q}$, you can always find a "gap" if you look closely enough (like $\sqrt{2}$ or $\pi$).

Section 2

The Completeness Axiom

The defining property of $\mathbb{R}$ is the **Least Upper Bound Property** (Completeness):

$$ \text{Every non-empty set of reals bounded above has a least upper bound in } \mathbb{R}. $$

Why this matters for CS

In Computer Science, we deal with floating point numbers which are technically a subset of the rationals. Because floating points are discrete, they cannot perfectly represent the continuum. Real Analysis provides the framework to understand the **error bounds** when we approximate the real world in a digital machine.

The Archimedean Property

Another crucial foundation: Given any real number $x$, there exists an integer $n$ such that $n > x$. This sounds obvious, but it proves that there are no "infinitely large" real numbers and that we can always reach any point by taking enough finite steps.

Section 3

Supremum & Infimum

Instead of "Maximum" (which might not exist for open intervals), we use **Supremum** ($\sup$).

  • **Supremum (Least Upper Bound):** The smallest possible upper bound.
  • **Infimum (Greatest Lower Bound):** The largest possible lower bound.

Example 1: Open Interval

For the set $S = \{ x \in \mathbb{R} : 0 < x < 1 \}$:

$\sup(S) = 1$ (even though $1 \notin S$).
$\inf(S) = 0$.

Example 2: Discrete Convergent Set

For the set $S = \{ 1, 1/2, 1/3, 1/4, \dots \}$:

$\sup(S) = 1$.
$\inf(S) = 0$.

Section 4

The ε-δ Definition

To be mathematically precise, we say $\lim_{x \to c} f(x) = L$ if for every $\epsilon > 0$, there exists a $\delta > 0$ such that:

$$ 0 < |x - c| < \delta \implies |f(x) - L| < \epsilon $$

The Game of ε and δ

Analogy: You are a manufacturer. A customer gives you a tolerance (error $\epsilon$) they can accept. You must find a precision ($\delta$) for your machine such that all output stays within that tolerance.

Crucial Distinction

Notice that the definition says $0 < |x - c|$. This means we don't care about the value of the function at $c$. We only care about what happens as we get "arbitrarily close".

Section 5

Python: Convergence & Cauchy

A sequence is **Cauchy** if its terms get arbitrarily close to *each other*. In $\mathbb{R}$, every Cauchy sequence converges. This is the "internal" version of completeness.

Python Convergence Testing 
import math

def sequence_a(n):
    # a_n = (2n + 1) / n -> Limits to 2
    return (2 * n + 1) / n

def test_limit(seq_func, L, epsilon):
    """
    Simulates checking the epsilon-N definition:
    For a given ε, find an N such that for all n > N, |a_n - L| < ε
    """
    found_n = -1
    for n in range(1, 100000):
        if abs(seq_func(n) - L) < epsilon:
            # We must check if it REMAINS within epsilon
            # (In this simple monotonic case, it does)
            found_n = n
            break
    return found_n

# Parameters
L = 2
epsilons = [0.1, 0.01, 0.001, 0.0001]

for e in epsilons:
    n_required = test_limit(sequence_a, L, e)
    print(f"For ε = {e}, required N is {n_required}")

# Result: As ε gets smaller, N must get larger.
