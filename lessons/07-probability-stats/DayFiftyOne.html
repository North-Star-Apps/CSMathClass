Section 1

The Digital Approximation

In Pure Math, we deal with $\mathbb{R}$ (Real Numbers). They are infinitely dense, continuous, and exact. $\pi$ is exactly $\pi$.

In Computer Science, we deal with $\mathbb{F}$ (Floating Point Numbers). They are finite, discrete, and riddled with holes. $\pi$ is `3.141592653589793`.

The Patriot Missile Failure (1991):
The Patriot missile system used 24-bit precision to store time in tenths of a second.
$1/10 = 0.00011001100..._2$ (Binary repeating decimal).
After 100 hours of operation, the rounding error accumulated to ~0.34 seconds.
For a missile traveling at Mach 5, 0.34 seconds is ~600 meters. The missile missed an incoming SCUD, resulting in 28 casualties.
Section 2

IEEE 754 Standard

How do we store "scientific notation" in bits? $$ Value = (-1)^S \times (1.M) \times 2^{E - Bias} $$

Double Precision (64-bit)

  • Sign (1 bit): +/-
  • Exponent (11 bits): Range ~10^{-308} to 10^{308}
  • Mantissa (52 bits): Precision ~15-17 decimal digits

Special Values

  • Infinity: Exp=All 1s, Mantissa=0. (e.g. 1/0)
  • NaN: Exp=All 1s, Mantissa!=0. (e.g. 0/0)
  • Subnormal: Exp=0. Allows tiny numbers near zero.

Analogy: The Broken Odometer
Imagine a car odometer that only has 8 digits. It can represent `99,999,999` miles or `0.0000001` miles.
But if you have `10,000,000` miles and drive `1` inch, the odometer doesn't move. The "1 inch" is lost because the scale is too large.
Floating point numbers have "dynamic range". The gaps between representable numbers get larger as the numbers get larger.
Section 3

Sources of Error

Machine Epsilon ($\epsilon$)

Machine Epsilon is the smallest number such that $1.0 + \epsilon \neq 1.0$. For 64-bit float, $\epsilon \approx 2.22 \times 10^{-16}$. Any change smaller than this relative to 1.0 is ignored.

Catastrophic Cancellation

Subtracting two large, nearly equal numbers is the most dangerous operation in numerics.

a = 1.23456789
b = 1.23456788
a - b = 0.00000001

We started with 9 significant digits. We ended with 1 significant digit. The rest of the bits in the result will be garbage/noise. This can destabilize an entire simulation.

Section 4

Stability & Conditioning

When solving a problem $f(x) = y$, two things can go wrong:

  1. Ill-Conditioned Problem: The problem itself is sensitive. A tiny change in input $x$ leads to a huge change in answer $y$. (e.g., finding roots of $(x-1)^{20}$).
  2. Unstable Algorithm: The problem is fine, but your method accumulates error.
Analogy: Balancing a Pencil
Balancing a sharp pencil on its tip is an Ill-Conditioned Problem. Even perfect physics leads to it falling.
Stacking bricks is a well-conditioned problem. But if you stack them using a trembling robot hand (Rounding Error), the pile might fall. That is an Unstable Algorithm.

Condition Number $\kappa$

Measures how much output error is amplified relative to input error. $$ \text{Relative Output Error} \approx \kappa \times \text{Relative Input Error} $$ If $\kappa$ is large ($10^{10}$), your answer is garbage because computers always have small input errors ($\epsilon$).

Section 5

Root Finding 101

There is no formula for roots of polynomials degree 5+ (Abel-Ruffini). We must use iterative algorithms to approximations.

Bisection Method

The "Binary Search" of math.
1. Find $a, b$ such that $f(a)$ and $f(b)$ have opposite signs.
2. Check midpoint $c = (a+b)/2$.
3. Replace $a$ or $b$ to keep the sign change.
4. Repeat.
Convergence: Linear (gain 1 bit of precision per step). Slow but robust.

Section 6

Newton's Method

If we know the derivative $f'(x)$, we can go faster. We approximate the curve as a tangent line and find where the line hits zero.

$$ x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)} $$

Convergence: Quadratic. The number of correct digits doubles every step (1, 2, 4, 8, 16...).
Risk: If $f'(x) \approx 0$, it shoots off to infinity (Instability).

Python IEEE 754 Inspector 
import struct

def float_digits(n):
    # Pack float into 8 bytes (double precision)
    packed = struct.pack('!d', n)
    # Unpack as integer to view bits
    integers = struct.unpack('!Q', packed)[0]
    bits = f"{integers:064b}"
    
    sign = bits[0]
    exponent = bits[1:12]
    mantissa = bits[12:]
    
    print(f"Value: {n}")
    print(f"Bits:  {sign} {exponent} {mantissa}")
    return bits

float_digits(0.1)
float_digits(0.2)
# Notice: 0.1 + 0.2 != 0.3 pattern
Python Newton's Method 
def newton_method(f, df, x0, tol=1e-10, max_iter=20):
    x = x0
    print(f"{'Iter':<5} {'x':<20} {'f(x)':<20}")
    print("-" * 50)
    
    for i in range(max_iter):
        y = f(x)
        if abs(y) < tol:
            print(f"Converged to {x} in {i} iterations.")
            return x
            
        dy = df(x)
        if dy == 0:
            print("Derivative zero. Failed.")
            return None
            
        x_new = x - y / dy
        print(f"{i:<5} {x:.10f}          {y:.5e}")
        x = x_new
        
    print("Did not converge.")
    return x

# Solve x^2 - 2 = 0 (Find sqrt(2))
newton_method(lambda x: x**2 - 2, lambda x: 2*x, 1.5)
