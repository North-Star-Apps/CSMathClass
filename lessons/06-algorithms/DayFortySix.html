Section 1

Weighted Graphs

In a standard BFS, every edge has "cost" 1. But in real life, roads have different lengths.

Weighted Graph: Each edge $(u, v)$ has a weight $w$.

Goal: Find path $P$ that minimizes $\sum w_i$.

Section 2

Dijkstra's Algorithm

The "Greedy" approach. It's BFS but with a Priority Queue instead of a normal Queue.

Steps:

  1. Set distance to Start = 0, others = $\infty$.
  2. Add (0, Start) to Min-Heap.
  3. Pop node $u$ with smallest distance.
  4. For each neighbor $v$:
    • If $dist[u] + weight(u,v) < dist[v]$:
    • Update $dist[v]$ (Relaxation).
    • Push $(dist[v], v)$ to Heap.

Complexity: $O((V+E) \log V)$.

Section 3

Negative Weights

Dijkstra fails if edge weights are negative (it assumes finding a path never increases cost).

Bellman-Ford Algorithm

Can handle negative weights.

Relax all edges $V-1$ times.

Slower: $O(V \times E)$.

Section 4

Python: Dijkstra

Implementing Dijkstra using `heapq`.

Python Shortest Path 
import heapq

def dijkstra(graph, start):
    # graph = {node: [(neighbor, weight), ...]}
    distances = {node: float('infinity') for node in graph}
    distances[start] = 0
    
    # Priority Queue: (distance, node)
    pq = [(0, start)]
    
    while pq:
        current_dist, current_node = heapq.heappop(pq)
        
        # Optimization: Skip if we found a shorter path already
        if current_dist > distances[current_node]:
            continue
            
        for neighbor, weight in graph[current_node]:
            distance = current_dist + weight
            
            # Relaxation Step
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(pq, (distance, neighbor))
                
    return distances

# Test
g = {
    'A': [('B', 1), ('C', 4)],
    'B': [('C', 2), ('D', 5)],
    'C': [('D', 1)],
    'D': []
}
print(dijkstra(g, 'A'))
# Output: {'A': 0, 'B': 1, 'C': 3, 'D': 4}
# Path A -> B -> C -> D cost is 1+2+1=4, better than A -> C -> D (5)
