Arrays & Dynamic Arrays

Array: Contiguous memory block with O(1) random access by index.
ARRAY OPERATIONS: Access by index: O(1) Search (unsorted): O(n) Insert at end: O(1) amortized (dynamic) Insert at middle: O(n) Delete at middle: O(n)

Dynamic arrays (ArrayList, vector) grow automatically, typically doubling capacity when full. Amortized O(1) append.

02

Linked Lists

SINGLY LINKED LIST: [data|next] ‚Üí [data|next] ‚Üí [data|null] DOUBLY LINKED LIST: null ‚Üê [prev|data|next] ‚áÑ [prev|data|next] ‚Üí null
LINKED LIST OPERATIONS: Access by index: O(n) Insert at head: O(1) Insert at tail: O(1) with tail pointer Insert at middle: O(n) to find + O(1) to insert Delete (by ref): O(1)

Trade-off: Arrays excel at random access; lists excel at insertion/deletion with a known position.

Stacks (LIFO)

Stack: Last-In-First-Out. Only access the top element.
STACK OPERATIONS (all O(1)): push(x) - Add to top pop() - Remove from top peek() - View top without removing isEmpty()- Check if empty

Use cases: Function call stack, undo operations, expression parsing, DFS traversal. 

// Balanced parentheses check
function isBalanced(s) {
    const stack = [];
    const pairs = {'(':')', '[':']', '{':'}'};
    for (let c of s) {
        if (c in pairs) stack.push(c);
        else if (')]}'.includes(c)) {
            if (!stack.length || pairs[stack.pop()] !== c)
                return false;
        }
    }
    return stack.length === 0;
}
04

Queues (FIFO)

Queue: First-In-First-Out. Add at back, remove from front.
QUEUE OPERATIONS (all O(1)): enqueue(x) - Add to back dequeue() - Remove from front front() - View front without removing isEmpty() - Check if empty

Use cases: BFS traversal, scheduling, buffering, printer queues.

Variants: Deque (double-ended), Priority Queue (comes next section with heaps).

05

Hash Tables

Hash Table: Key-value store with O(1) average lookup, insert, delete.
HASH TABLE: key ‚Üí hash(key) ‚Üí index ‚Üí bucket COLLISION RESOLUTION: 1. Chaining: Each bucket is a linked list 2. Open Addressing: Probe for next empty slot - Linear probing: h(k) + 1, h(k) + 2, ... - Quadratic probing: h(k) + 1¬≤, h(k) + 2¬≤, ...
HASH TABLE COMPLEXITY: Average Worst Search: O(1) O(n) Insert: O(1) O(n) Delete: O(1) O(n) Worst case = all keys collide

Load factor: n/m (items/buckets). Rehash when load factor exceeds threshold (~0.75).
