Complexity Analysis (Big O)

Big O Hierarchy (slowest to fastest):
$$ O(n!) \gt O(2^n) \gt O(n^2) \gt O(n \log n) \gt O(n) \gt O(\log n) \gt O(1) $$
Complexity Name Example
$O(1)$ Constant Array access, hash lookup
$O(\log n)$ Logarithmic Binary search
$O(n)$ Linear Single loop, find max
$O(n \log n)$ Linearithmic Merge sort, heap sort
$O(n^2)$ Quadratic Nested loops, bubble sort
$O(2^n)$ Exponential Subsets, naive recursion
üèÉ Simple: The Running Track Analogy
Big O is like asking "how many laps around a track do I need?" $O(1)$ is standing still‚Äîno matter how big the stadium, you don't move. $O(n)$ is one lap per person. $O(n^2)$ is making every person run a lap for every other person‚Äîcrowds get exhausting fast!

Graph Algorithms

Traversal

  • BFS: Level-by-level, uses Queue. Good for shortest path (unweighted).
  • DFS: Go deep first, uses Stack/recursion. Good for connectivity, cycles.
  • Both are $O(V + E)$ time and space.

Shortest Paths

Algorithm Complexity Use Case
BFS $O(V + E)$ Unweighted graphs
Dijkstra $O(E \log V)$ Non-negative weights
Bellman-Ford $O(VE)$ Negative weights

Minimum Spanning Tree

  • Prim's: Greedy, grow tree from a vertex. $O(E \log V)$
  • Kruskal's: Sort edges, use Union-Find. $O(E \log E)$
üó∫Ô∏è Simple: The Maze Analogy
BFS is like water flooding a maze‚Äîit spreads evenly in all directions, finding the shortest path first. DFS is like a curious explorer who goes down one corridor as far as possible before backtracking. Both explore everything, just in different orders!

Dynamic Programming

The Two Key Properties:
  • Optimal Substructure: Solution can be built from solutions to subproblems
  • Overlapping Subproblems: Same subproblems solved multiple times

Classic DP Problems

Problem Complexity Pattern
Fibonacci $O(n)$ 1D array
0/1 Knapsack $O(nW)$ 2D: items √ó capacity
Longest Common Subsequence $O(mn)$ 2D: strings
Coin Change $O(n \cdot amount)$ 1D array
üß† Simple: The Notebook Analogy
DP is like solving homework and writing answers in a notebook. When you see the same subproblem again, you just look it up instead of re-solving. Memoization = writing answers as you go (top-down). Tabulation = filling in a table systematically (bottom-up).
