Searching

LINEAR SEARCH: O(n) Check each element sequentially BINARY SEARCH: O(log n) Requires sorted array Repeatedly halve search space
function binarySearch(arr, target) {
    let lo = 0, hi = arr.length - 1;
    while (lo <= hi) {
        const mid = Math.floor((lo + hi) / 2);
        if (arr[mid] === target) return mid;
        if (arr[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

Divide & Conquer

Pattern: 1. DIVIDE into subproblems ‚Üí 2. CONQUER recursively ‚Üí 3. COMBINE solutions
EXAMPLES: Merge Sort: Split ‚Üí Sort halves ‚Üí Merge Quick Sort: Partition ‚Üí Sort parts Binary Search: Check mid ‚Üí Search half Karatsuba: Multiply large numbers faster
// Merge Sort
function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));
    return merge(left, right);
}

function merge(l, r) {
    const result = [];
    while (l.length && r.length) {
        result.push(l[0] <= r[0] ? l.shift() : r.shift());
    }
    return [...result, ...l, ...r];
}

Greedy Algorithms

Greedy: Make locally optimal choice at each step, hoping for global optimum.
WHEN GREEDY WORKS: 1. Greedy choice property: local optimum leads to global 2. Optimal substructure: optimal solution contains optimal sub-solutions CLASSIC PROBLEMS: ‚Ä¢ Activity Selection: Pick non-overlapping activities ‚Ä¢ Huffman Coding: Build optimal prefix-free codes ‚Ä¢ Fractional Knapsack: Take items by value/weight ratio ‚Ä¢ Coin Change (some denominations): Minimum coins
// Activity Selection (greedy by end time)
function activitySelection(activities) {
    activities.sort((a, b) => a.end - b.end);
    const selected = [activities[0]];
    let lastEnd = activities[0].end;
    for (let i = 1; i < activities.length; i++) {
        if (activities[i].start >= lastEnd) {
            selected.push(activities[i]);
            lastEnd = activities[i].end;
        }
    }
    return selected;
}
