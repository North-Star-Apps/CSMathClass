Clean Code Principles

Core: Code is read more than written. Optimize for clarity.

Naming: Use intention-revealing names. Functions use verbs (getUser), classes use nouns (UserRepository), booleans use predicates (isValid).

FUNCTIONS SHOULD: ‚Ä¢ Be small (<20 lines) ‚Ä¢ Do one thing well ‚Ä¢ Have few arguments (0-2 ideal) ‚Ä¢ Follow command-query separation

SOLID Principles:

  • S - Single Responsibility
  • O - Open/Closed
  • L - Liskov Substitution
  • I - Interface Segregation
  • D - Dependency Inversion
02

Testing Strategies

TESTING PYRAMID: /\ E2E (few) / \ Integration (some) /----\ Unit (many)

Follow AAA pattern: Arrange ‚Üí Act ‚Üí Assert

function test_add() {
    // Arrange
    const calc = new Calculator();
    // Act
    const result = calc.add(2, 3);
    // Assert
    assert(result === 5);
}

Test: Happy path, edge cases (empty, null, max), error conditions, boundary cases.

03

Systematic Debugging

SCIENTIFIC METHOD: 1. OBSERVE - Reproduce reliably 2. HYPOTHESIZE - About the cause 3. PREDICT - What if hypothesis is true 4. EXPERIMENT - Test prediction 5. ANALYZE - Refine and repeat

Techniques: Binary search (halve area), rubber duck (explain aloud), print debugging, breakpoints, git bisect.

Common bugs: Off-by-one, null access, unexpected mutation, wrong operator.
