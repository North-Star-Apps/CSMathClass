Section 1

Graph Basics

Graph G = (V, E): V = vertices (nodes), E = edges (connections).

Terminology

GRAPH VOCABULARY: |V| = n ‚Äî number of vertices |E| = m ‚Äî number of edges Directed: edges have direction (u ‚Üí v) Undirected: edges are bidirectional Weighted: edges have values (costs/distances) Dense: m ‚âà n¬≤ (many edges) Sparse: m ‚âà n (few edges)

Special graphs

  • Tree: Connected graph with n-1 edges (no cycles)
  • DAG: Directed Acyclic Graph (topological order exists)
  • Complete: Every vertex connected to all others
  • Bipartite: Vertices split into two groups, edges only between groups
Section 2

Adjacency Matrix

n √ó n matrix: A[i][j] = 1 if edge (i,j) exists, 0 otherwise.
ADJACENCY MATRIX EXAMPLE: 0 1 2 3 ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ 0 ‚îÇ 0 1 0 1 1 ‚îÇ 1 0 1 0 2 ‚îÇ 0 1 0 1 3 ‚îÇ 1 0 1 0 Edge (0,1) exists ‚Üí A[0][1] = 1

Complexity

ADJACENCY MATRIX: Space: O(n¬≤) Check edge: O(1) Find neighbors: O(n) Add edge: O(1) Add vertex: O(n¬≤) ‚Äî resize matrix
When to use: Dense graphs OR when you need O(1) edge lookup. Wastes space for sparse graphs.
Section 3

Adjacency List

Array of lists: adj[v] = list of vertices connected to v.
ADJACENCY LIST EXAMPLE: 0 ‚Üí [1, 3] 1 ‚Üí [0, 2] 2 ‚Üí [1, 3] 3 ‚Üí [0, 2] Same graph, much less space!

Complexity

ADJACENCY LIST: Space: O(n + m) Check edge: O(degree) ‚âà O(n) worst Find neighbors: O(degree) Add edge: O(1) Add vertex: O(1)
PythonBuilding an adjacency list
# Using dictionary of lists
graph = {
    0: [1, 3],
    1: [0, 2],
    2: [1, 3],
    3: [0, 2]
}

# Add edge (u, v)
graph[u].append(v)
graph[v].append(u)  # for undirected
Best for: Most algorithms (BFS, DFS, Dijkstra). The standard choice for sparse graphs.
Section 4

Edge List

List of edges: Each edge stored as (u, v) or (u, v, weight).
EDGE LIST EXAMPLE: [(0,1), (0,3), (1,2), (2,3)] With weights: [(0,1,5), (0,3,2), (1,2,3), (2,3,1)]

Complexity

EDGE LIST: Space: O(m) Check edge: O(m) ‚Äî linear scan Find neighbors: O(m) Add edge: O(1) Best for: Kruskal's MST (sort edges by weight)
