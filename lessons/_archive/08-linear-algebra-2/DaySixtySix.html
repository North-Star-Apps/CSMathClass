Section 1

The Downhill Hike

Imagine you are blindfolded on a foggy mountain and want to reach the valley floor.

  • You feel the ground around you to find the steepest downward slope.
  • You take a small step in that direction.
  • You repeat until you can't go down anymore.

In math, the "mountain" is a **Cost Function** $J(\theta)$, and we want to find the parameters $\theta$ that minimize it.

Section 2

The Gradient ($\nabla$)

The Gradient is a vector of partial derivatives. It points in the direction of the **steepest ascent**.

$$ \nabla f(x, y) = \begin{bmatrix} \frac{\partial f}{\partial x} \\ \frac{\partial f}{\partial y} \end{bmatrix} $$

To go down, we must walk in the direction of the negative gradient: $-\nabla f$.

Section 3

The Update Rule

We update our parameters iteratively:

$$ \theta_{new} = \theta_{old} - \alpha \cdot \nabla J(\theta) $$

Where $\alpha$ is the **Learning Rate** (the size of our step).

Section 4

Learning Rates ($\alpha$)

Step Size Result
**Too Small** Takes forever to converge. Might get stuck in local minima.
**Too Big** Overshoots the valley. Might diverge and never find the minimum.
**Just Right** Smooth path to the global minimum.
Section 5

Python: Minimizing a Function

Minimizing $f(x) = x^2$ using Gradient Descent.

Python Basic GD 
def f(x):
    return x**2

def grad(x):
    return 2*x

# Hyperparameters
x = 10 # Starting point
learning_rate = 0.1
iterations = 50

print(f"Starting at x = {x}")

for i in range(iterations):
    # Calculate gradient
    g = grad(x)
    
    # Update x
    x = x - learning_rate * g
    
    if i % 5 == 0:
        print(f"Iteration {i}: x = {x:.4f}, f(x) = {f(x):.4f}")

print(f"\nFinal x: {x:.4f} (Goal: 0)")
