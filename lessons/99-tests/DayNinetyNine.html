Section 2

Test Strategy: Triage

In CS, when a server is overloaded, we dropload requests. You must do the same. Do not spend 20 minutes on one hard integral while missing 10 easy logic questions.

The O(N) Scan:
1. Read the question.
2. If you know how to solve it immediately (O(1)), solve it.
3. If you need to think (O(N)), mark it and skip.
4. Only return to skipped questions after you have seen EVERY question.

Analogy: The Triage Nurse
A nurse in the ER assesses patients.
Patient A has a heart attack (Critical/High Value).
Patient B has a papercut (Low Value).
Patient C has a complex, undiagnosed fever (Time Sink).
You treat cardiac arrests first. You treat papercuts quickly. You do NOT spend 4 hours diagnosing the fever while the heart attack patient dies.
Skip the Time Sinks.
Section 3

The Math of Guessing

Should you guess? Let's use Expected Value (EV) to decide.
Assume +1 point for correct, -0.25 for incorrect (common penalty).

Python Guessing Simulation 
def expected_value(correct_reward, wrong_penalty, options_remaining):
    prob_correct = 1 / options_remaining
    prob_wrong = (options_remaining - 1) / options_remaining
    
    ev = (prob_correct * correct_reward) + (prob_wrong * wrong_penalty)
    return ev

# Scenario 1: Blind Guess (4 options)
ev_blind = expected_value(1.0, -0.25, 4)
print(f"Blind Guess EV: {ev_blind:.2f}") 
# (0.25 * 1) + (0.75 * -0.25) = 0.25 - 0.1875 = +0.06
# Even fully blind, it's slightly positive!

# Scenario 2: Eliminate 1 (3 options)
ev_elim1 = expected_value(1.0, -0.25, 3)
print(f"Eliminate 1 EV: {ev_elim1:.2f}")
# (0.33 * 1) + (0.66 * -0.25) = 0.33 - 0.165 = +0.16

# Scenario 3: Eliminate 2 (Coin flip)
ev_elim2 = expected_value(1.0, -0.25, 2)
print(f"Eliminate 2 EV: {ev_elim2:.2f}")
# (0.5 * 1) + (0.5 * -0.25) = 0.5 - 0.125 = +0.375 !!
Conclusion: If there is NO penalty for wrong answers, always guess. If there IS a penalty, you should still guess if you can eliminate even one option. Leaving a question blank has an EV of 0. Guessing almost always has EV > 0.
