Why Function Types Matter in CS

Every algorithm you write has a performance characteristic that can be described as a function. Understanding polynomials, exponentials, and logarithms is the foundation of Big-O analysis.

The Core Principle: Functions are the language of relationship. Polynomials model physics, rational functions model rates, piecewise functions model logic, and exponentials determine whether your code will finish before the heat death of the universe.
‚òï Simple: A function is like a coffee machine. You put in beans (input x), and it gives you coffee (output y). The "type" of function describes HOW the machine transforms beans to coffee. A polynomial is a predictable grinder. An exponential is a machine that doubles your coffee with every button press‚Äîsounds great until you're drowning in espresso!

Where you'll use this

  • Algorithm Analysis: O(n¬≤) vs O(n log n) are polynomial comparisons
  • Graphics: Bezier curves are polynomials, color gamma is exponential
  • Machine Learning: Activation functions, loss landscapes
  • Cryptography: Discrete logarithms underpin RSA and Diffie-Hellman
  • Database Indexing: B-trees give O(log n) lookups
PythonFunction types determine algorithm scalability
# These functions ALL return ~1000 for n=1000
# But watch what happens at n=1,000,000:

def logarithmic(n): return math.log2(n) * 100  # ~2000

def quadratic(n):   return n ** 2       # 1,000,000,000,000
def exponential(n): return 2 ** n       # 10^301029 (UNIVERSE ENDS)

# This is why we obsess over Big-O!

üèóÔ∏è Simple: A polynomial is like building a LEGO tower. Each term (like 3x¬≤) is a block. The "degree" is the height of your tallest tower. Just like taller towers dominate the skyline, higher-degree terms dominate when x gets big.

Key Properties

  • Degree: The highest power determines end behavior and max roots
  • Roots: A degree-n polynomial has at most n real roots

  • End behavior: Even degree = both ends same direction; Odd degree = opposite

The "Power Level" Rule

Dominance Law: As x ‚Üí ‚àû, the highest-degree term dominates completely. All other terms become noise.
f(x) = 5x¬≥ + 1000x¬≤ + 999999x + 10000000 For large x: f(x) ‚âà 5x¬≥ At x = 1000: 5x¬≥ = 5,000,000,000 1000x¬≤ = 1,000,000,000 The x¬≥ term is 5√ó bigger!
üöÄ Simple: In a race to infinity, a Degree-3 rocket ALWAYS beats a Degree-2 rocket, no matter how much fuel the Degree-2 starts with. This is why O(n¬≥) algorithms will eventually be beaten by O(n¬≤) algorithms, given enough data.

Rational Functions

Definition

R(x) = P(x) / Q(x) A ratio of two polynomials where Q(x) ‚â† 0
üöß Simple: A rational function is like a road with invisible fences. The denominator creates "no-go zones" where division by zero would occur. The graph gets infinitely close to these fences but never crosses them‚Äîthese are asymptotes!

Types of Asymptotes

Vertical Asymptotes: Occur at x-values where denominator = 0 (and numerator ‚â† 0). The graph shoots to ¬±‚àû.
Horizontal Asymptotes: The y-value the function approaches as x ‚Üí ¬±‚àû. Compare degrees of numerator and denominator.
For R(x) = P(x)/Q(x): deg(P) < deg(Q): H.A. at y=0 deg(P)=deg(Q): H.A. at y=(lead coeff of P)/(lead coeff of Q) deg(P)> deg(Q): No H.A. (oblique/slant asymptote instead)

Holes vs Asymptotes

üï≥Ô∏è Simple: When both numerator and denominator are zero at the same x-value, you get a "hole" (removable discontinuity), not an asymptote. It's like a pothole in the road‚Äîyou could fill it in, but someone left it empty.
f(x) = (x¬≤ - 4) / (x - 2) = (x+2)(x-2) / (x-2) = x + 2 But only for x ‚â† 2! At x = 2, there's a HOLE at point (2, 4)

Logarithm Laws: log(xy) = log(x) + log(y) (product ‚Üí sum) log(x/y) = log(x) - log(y) (quotient ‚Üí difference) log(x‚Åø) = n¬∑log(x) (power ‚Üí multiply)
üå≤ Simple: If exponentials are about "how big does it get?", logarithms answer "how many times did we split?" log‚ÇÇ(1024) = 10 because you need to halve 1024 exactly 10 times to get to 1. This is why binary search is O(log n)‚Äîeach step cuts the problem in half!
PythonLogarithms in algorithm analysis

def binary_search_steps(n):
    return math.ceil(math.log2(n))

print("1000 elements:", binary_search_steps(1000))     # 10 steps
print("1,000,000 elements:", binary_search_steps(1_000_000))  # 20 steps!

# Double the data, add ONE step. That's the power of O(log n).

Common Gotchas

1. Degree ‚â† Coefficient
3x¬≤ has degree 2, not 3. The degree is the exponent, not the multiplier.
2. Logarithm of Zero/Negative
log(0) and log(-5) are undefined in real numbers. Always check domain!
3. Asymptote ‚â† Never Cross
Horizontal asymptotes CAN be crossed by the function; vertical asymptotes cannot.
4. Base Matters for Logs
log‚ÇÇ(8) = 3, but log‚ÇÅ‚ÇÄ(8) ‚âà 0.9. The base changes the answer completely!
5. n¬≤ vs 2‚Åø
These look similar but are DRAMATICALLY different. n¬≤ = 100 for n=10; 2‚Åø = 1024 for n=10. At n=20: n¬≤=400, 2‚Åø=1,048,576!
