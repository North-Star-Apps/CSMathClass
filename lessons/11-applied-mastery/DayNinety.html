Section 1

The Relation: A Set of Tuples

In mathematics, a **Relation** is a subset of the Cartesian product of several sets. In databases, we call this a **Table**.

Each row in the table is a **Tuple**. Each column is an **Attribute**. Crucially, in strict Relational Algebra, a relation is a **Set**, meaning there are no duplicate rows.

Section 2

The Foundation Operators

There are six fundamental operators in relational algebra:

  • **Selection ($\sigma$):** Filters rows (The `WHERE` clause). Example: $\sigma_{age > 21}(Users)$.
  • **Projection ($\pi$):** Filters columns (The `SELECT` clause). Example: $\pi_{name, email}(Users)$.
  • **Union ($\cup$):** Combines two tables with the same schema.
  • **Difference ($-$):** Rows in A but not in B.
  • **Cartesian Product ($\times$):** Every possible pair of rows.
  • **Rename ($\rho$):** Changes headers.

Section 3

The Join $(\bowtie)$

The **Natural Join** is arguably the most important operation. It combines two relations based on common attributes, filtering out the "useless" pairs from a Cartesian product.

Mathematically: $R \bowtie S = \pi_{attrs}( \sigma_{R.id = S.id}(R \times S) )$.

Section 4

Algebraic Optimization

Because these are mathematical operators, we can use **Algebraic Laws** to optimize queries. For example:

Selection Pushing

$\sigma_p(R \bowtie S) \equiv (\sigma_p R) \bowtie S$ (if the predicate $p$ only involves $R$).

It is always faster to filter the rows **before** doing the expensive join!

Section 5

Python: Implementing a Hash Join

How does a database engine actually "Join" two lists of data efficiently? One common method is the **Hash Join** ($O(N+M)$).

Python Inner Join Implementation 
# Two 'tables' represented as lists of dicts
users = [
    {"id": 1, "name": "Alice"},
    {"id": 2, "name": "Bob"},
    {"id": 3, "name": "Charlie"}
]

orders = [
    {"user_id": 1, "product": "Laptop"},
    {"user_id": 1, "product": "Mouse"},
    {"user_id": 2, "product": "Keyboard"},
]

def hash_join(table_a, table_b, key_a, key_b):
    # 1. Build Phase: Hash the smaller table
    hash_map = {}
    for row in table_a:
        hash_map.setdefault(row[key_a], []).append(row)
    
    # 2. Probe Phase: Match rows from the larger table
    result = []
    for row in table_b:
        if row[key_b] in hash_map:
            for match in hash_map[row[key_b]]:
                # Merge dicts (the 'Natural Join' effect)
                result.append({**match, **row})
    return result

joined = hash_join(users, orders, "id", "user_id")
for row in joined:
    print(row)
