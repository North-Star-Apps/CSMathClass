Distributed Systems & Consensus

CAP Theorem: In a distributed system, you can only guarantee 2 of 3:
  • Consistency: All nodes see the same data
  • Availability: Every request gets a response
  • Partition Tolerance: System works despite network failures

Consensus Algorithms

Paxos Propose ‚Üí Accept ‚Üí Learn Foundation for others

CAP is like a restaurant chain. Consistency = all locations have the same menu. Availability = never "closed for updates." Partition tolerance = keeps working even if phones between locations go down. Pick 2: you can't have all 3 perfectly!

Database Theory

Relational Algebra Operators:
  • Selection ($\sigma$): Filter rows (WHERE clause)
  • Projection ($\pi$): Select columns (SELECT clause)
  • Join ($\bowtie$): Combine tables on matching keys
  • Union ($\cup$), Difference ($-$): Set operations

Query Optimization

Key rule: Push selections down!

$$ \sigma_p(R \bowtie S) \equiv (\sigma_p R) \bowtie S $$

Filter before joining to reduce intermediate result sizes.

Financial Mathematics

Black-Scholes Equation:
$$ \frac{\partial V}{\partial t} + \frac{1}{2}\sigma^2 S^2 \frac{\partial^2 V}{\partial S^2} + rS\frac{\partial V}{\partial S} - rV = 0 $$
  • $V$ = option value, $S$ = stock price
  • $\sigma$ = volatility, $r$ = risk-free rate

Key Concepts

  • Risk-Neutral Pricing: Price as expected value under Q-measure
  • Delta Hedging: Neutralize price risk with $\Delta = \frac{\partial V}{\partial S}$
  • Monte Carlo: Simulate paths for complex derivatives
